Tier 2 — control flow

Making code decide and repeat: if/else, the comparison operators, the for...of loop, and the ternary ? :.

Your own file gives a heads-up: channelSolidNexus.ts has no else and no ternary — on purpose. It uses guard clauses (bail early) and ?? instead. Learn the tools; notice the choice.

Run anything by pasting into scratch.js and running node scratch.js.

CARD 2.1 — if / else and the guard clause

What it does: run a block only when a condition is truthy. else is the fallback.

const hour = 14;
if (hour < 12) {
  console.log("morning");
} else {
  console.log("afternoon"); // → afternoon
}

The guard clause — what your file actually uses. Handle the odd case first and return out, so the rest runs un-nested:

function greet(name) {
  if (!name) return "hi, stranger"; // bail early
  return "hi, " + name;             // normal path, no else
}
console.log(greet(""));     // → hi, stranger
console.log(greet("Cody")); // → hi, Cody

When the body is a single statement, you can drop the { } — if (!name) return "..."; is one line.

You've seen it: if (existing) return existing; = "already loading? hand it back, stop." if (ids.length > 0) return; = "already have channels? nothing to do." That early return is why the whole file has zero else.

  return "has " + count;

console.log(status(3));
CARD 2.2 — comparisons: === !== < > !

What it does: produce a true/false to feed an if. The big rule lives here.

console.log(1 === 1);      // → true   equal? (value AND type)
console.log("a" !== "b");  // → true   not equal?
console.log(5 > 3);        // → true
console.log(!true);        // → false  ! flips a boolean

The one rule: always ===, never ==. Double-equals converts types behind your back; triple-equals is honest:

console.log(0 == "");    // → true   (== converts — chaos)
console.log(0 === "");   // → false  (=== honest — number isn't string)
console.log(1 == "1");   // → true   (== converting "1" to 1)
console.log(1 === "1");  // → false  (number vs string)

This codebase uses === / !== only — the linter rejects ==. Treat == as a bug.

You've seen it: if (this.state.activeChannelId === id) return; = "already on this channel? do nothing." if (!communityIds.includes(channel.id)) = "if it does NOT already include this id, add it."

Your turn — which print? 
if (2 === 2) console.log("A");
if (2 == "2") console.log("B");
if (2 === "2") console.log("C");
if (!0) console.log("D");
CARD 2.3 — the for...of loop

What it does: run a block once per item in an array. The name becomes each item in turn.

const names = ["general", "random", "voice"];
for (const name of names) {
  console.log(name);
}
// → general
// → random
// → voice

Read it as "for each name of names." The older for loop appears when you need the index:

for (let i = 0; i < 3; i = i + 1) {
  console.log(i); // → 0, then 1, then 2
}

Three parts: start (i = 0), test (i < 3), step (i = i + 1). Reach for for...of unless you need i.

You've seen it: for (const raw of sorted) in setChannels. Many "loops" here hide inside .map() / .filter() (tier 4) — loops in disguise — but for...of is the visible baseline.

Your turn — what prints? 
const nums = [10, 20, 30];
let total = 0;
for (const n of nums) {
  total = total + n;
}
console.log(total);
CARD 2.4 — the ternary ? :

What it does: an if/else that produces a value. Shape: condition ? valueIfTrue : valueIfFalse.

const age = 20;
const label = age >= 18 ? "adult" : "minor";
console.log(label); // → adult

That one line replaces a four-line if/else that assigns label. The ternary is for when you want a value, not a block of actions.

You've seen it: not in the nexus — it prefers guard clauses and ??. But in the UI (tier 8), ternaries fill JSX, like {loading ? "Loading…" : content}. Learn the shape now so it's not alien then.

Your turn — what prints? 
const n = 7;
const kind = n % 2 === 0 ? "even" : "odd"; // 7 % 2 is 1
console.log(kind);

Tier 2 checkpoint

  • a guard clause if (x) return; reads as "bail out early"
  • you reach for === automatically and treat == as a mistake
  • for...of reads as "once per item"
  • a ternary reads as "an if/else that hands back a value"
