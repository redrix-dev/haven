CARD 4.1 — array transformers: .map .filter .find .includes

What it does: tier 3 taught that these take a function. Now: what each gives back. Headline — none of them change the original.

const nums = [1, 2, 3, 4];

console.log(nums.map((n) => n * 10));   // → [10,20,30,40]  new, SAME length
console.log(nums.filter((n) => n > 2)); // → [3, 4]         new, maybe SHORTER
console.log(nums.find((n) => n > 2));   // → 3              FIRST match (one item)
console.log(nums.includes(3));          // → true           a yes/no

console.log(nums); // → [1,2,3,4]   ← untouched by all of it
  • .map → new array, same length, each transformed
  • .filter → new array, possibly shorter, items that passed
  • .find → one item (first match) or undefined
  • .includes → true/false

(A fourth sibling, .reduce, folds a list into one value — nums.reduce((total, n) => total + n, 0) is 10. Not in your nexus, but now it won't startle you.)

You've seen it: channels.map((channel) => channel.id) (channels → ids), ids.filter((channelId) => channelId !== id) (drop one), communityIds.includes(channel.id) (already here?).

CARD 4.2 — the spread ...

What it does: unpacks an array or object into a new one. The codebase's answer to value-vs-reference: don't mutate the shared box, build a fresh copy.

const a = [1, 2];
const b = [...a, 3];   // a's items, then 3
console.log(b); // → [1, 2, 3]
console.log(a); // → [1, 2]   ← original untouched

const base = { name: "general", kind: "text" };
const renamed = { ...base, name: "lobby" }; // copy, then override name
console.log(renamed); // → { name: 'lobby', kind: 'text' }
console.log(base);    // → { name: 'general', kind: 'text' }  ← untouched

Tier 1: list2 = list1 shared one array. [...list1] makes a genuine copy, so changes don't leak. That's the whole point.

Order matters: later keys win. { ...base, name: "lobby" } overrides; { name: "lobby", ...base } lets base win back.

You've seen it: [...communityIds, channel.id] (copy + append → Solid sees a change), { ...group, channelIds: ... } (copy a group with fresh ids), and [...channels].sort(...) — .sort mutates in place, so you spread into a copy first to protect the original.

Your turn — what prints? 
const original = { a: 1, b: 2 };
const updated = { ...original, b: 99, c: 3 };
console.log(updated);
console.log(original);
CARD 4.3 — destructuring

What it does: pull pieces out of an array or object into their own names, in one line.

const pair = [10, 20];
const [first, second] = pair;   // by POSITION
console.log(first, second); // → 10 20

const channel = { id: "abc", name: "general", kind: "text" };
const { name, kind } = channel; // by NAME
console.log(name, kind); // → general text
  • arrays: const [first, second] = pair → position decides.
  • objects: const { name } = channel → the name must match the key.

You've seen it: const [state, setState] = createStore(...) — createStore hands back a two-item array [store, setter]; this names both at once, by position. The names are yours — const [s, set] = ... works identically.

Your turn — what prints? 
const box = { hairpins: 4, gatorade: 3 };
const { hairpins } = box;
console.log(hairpins);

const [a, b, c] = [100, 200, 300];
console.log(b);
CARD 4.4 — ?. and ??

What it does: handle "this might be missing" without crashing.

const user = { name: "Cody" };
console.log(user.address?.city); // → undefined  (no crash)
// without ?. , user.address.city CRASHES

console.log(null ?? "fallback"); // → fallback
console.log(0 ?? "fallback");    // → 0  (0 is real, stays)
  • ?. = "if the thing before me is null/undefined, stop and give undefined instead of throwing."
  • ?? = "use the left; only if it's null/undefined, use the right."

The ?? vs || trap: || falls back on any falsy value, including 0 and "" (often real data). ?? only on null/undefined.

console.log(0 || "fallback"); // → fallback  (treats 0 as missing — usually wrong)
console.log(0 ?? "fallback"); // → 0         (keeps real 0 — usually right)

You've seen it: this.state.byCommunity[communityId] ?? [] (the list, or empty), existing ?? groupState.collapsedGroupIds (keep, or fall back), and if (!ids?.length) in the shared selector (missing OR length 0).

Your turn — what prints? 
const data = { count: 0 };
console.log(data.count ?? 99);   // is 0 null/undefined?
console.log(data.missing ?? 99);
console.log(data.nested?.deep);

Tier 4 checkpoint

  • .map = new array same length, .filter = maybe shorter, .find = one item, .includes = true/false — none touch the original
  • spread ... reads as "make a fresh copy" (arrays and objects)
  • destructuring reads as "pull pieces into names" — [by position], {by name}
  • ?. = reach in safely; ?? = fall back only on null/undefined
